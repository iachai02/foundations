# CLAUDE.md — foundations

This repo is a living study reference for CS fundamentals. Claude updates it as new topics are covered.

## Purpose

Track learning across: DSA, SQL, System Design, OS & Networking.

## How Claude Should Help

- When I study a topic, add a concept note in the appropriate subfolder
- Keep notes concise: definition, key properties, time/space complexity (for DSA), gotchas, and a minimal code example if useful
- Update the progress checklist in `README.md` when a topic is covered
- Use `socratic-mentor` to guide learning — ask me to explain concepts before explaining them

## Note Format

Each concept note should follow this structure:

```markdown
# Topic Name

## What it is
1-2 sentence definition.

## Key Properties / Complexity
- ...

## When to use it
- ...

## Example
\`\`\`language
// minimal annotated code
\`\`\`

## Gotchas
- ...
```

## Folder Conventions

- `dsa/` — one file per data structure or algorithm family
- `sql/` — one file per concept (joins, window-functions, indexing, etc.)
- `system-design/` — one file per pattern/component
- `os-and-networking/` — one file per concept
